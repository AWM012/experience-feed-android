# 3 分钟演示录屏脚本

## 0:00 - 0:20 项目介绍

打开应用首页，说明这是参考抖音「经验」栏实现的双列图片流作业。

需要点到的功能：

- 双列瀑布流列表。
- 默认上海的实时天气，以及修改天气地点。
- 图片标题、描述、日期、大小、位置。
- OkHttp 网络图片请求。
- 图片缓存复用。

## 0:20 - 1:00 首页功能演示

展示首页滚动效果：

- 顶部 tab 中「经验」处于选中状态。
- 天气卡片展示上海的温度、天气、湿度、风速和更新时间。
- 点击「修改地点」，输入北京或杭州，展示天气刷新。
- 图片以左右两列展示。
- 每张卡片有不同高度，形成瀑布流。
- 首次进入时，图片加载来源会显示「网络下载」。

可以口播：

> 首页不是写死图片资源，而是通过 Retrofit 声明下载接口，由 OkHttp 真正执行网络请求。请求成功后会把图片写入应用缓存目录，同时更新图片大小和加载来源。

> 天气功能也使用 Retrofit。因为天气接口需要经纬度，所以我先根据城市名查询经纬度，再请求实时天气，Gson 会把返回的 JSON 自动转换成 Java 对象。

## 1:00 - 1:35 详情页演示

点击任意图片卡片进入详情页：

- 展示大图。
- 展示标题和描述。
- 展示图片日期、图片大小、拍摄位置。
- 展示本地缓存文件路径。
- 展示加载来源。

可以口播：

> 详情页和首页没有各自维护下载逻辑，而是复用同一个 ImageRepository，所以只要首页已经下载过，详情页会优先命中内存缓存或磁盘缓存。

## 1:35 - 2:10 缓存命中演示

操作方式：

1. 返回首页。
2. 关闭应用进程。
3. 重新打开应用。
4. 观察已经下载过的图片显示「磁盘缓存」。

可以口播：

> 这里可以看到，重新启动后没有重复下载已经缓存的图片，而是直接从应用缓存目录解码本地文件。

## 2:10 - 2:40 代码结构说明

切到 Android Studio，简要展示几个核心文件：

- `MainActivity.java`：双列列表和卡片点击。
- `DetailActivity.java`：大图详情和图片信息。
- `ImageDownloadService.java`：Retrofit 图片下载接口。
- `ImageRepository.java`：OkHttp 客户端、拦截器、线程池和多级缓存。
- `WeatherService.java`：Retrofit 城市搜索和天气接口。
- `WeatherRepository.java`：串联城市搜索与实时天气请求。
- `FeedData.java`：图片数据源。

可以口播：

> 我把 UI 展示和图片加载拆开，Activity 只负责页面状态，缓存和网络请求都集中在 ImageRepository 中，方便复用和测试。

## 2:40 - 3:00 技术难点与优化总结

总结 3 点：

- 网络请求不能放在主线程，所以使用线程池处理请求和文件 IO。
- 重复下载影响体验，所以加了内存缓存和磁盘缓存。
- 弱网和异常响应需要兜底，所以对 HTTP 错误、空响应、解码失败都做了错误回调。

最后说明：

> 后续如果继续完善，我会把列表替换为 RecyclerView，并增加缓存过期策略和失败占位图。
